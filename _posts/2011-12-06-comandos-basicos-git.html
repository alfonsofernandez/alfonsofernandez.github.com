Iniciar un repositorio

git init mirepo.git Crea una carpeta en el directorio actual con el nombre mirepo.git con un subdirectorio oculto llamado .git/ donde están todos los ficheros del repositorio.

Añadir todo a el stagging area.

git add . Añade todos los archivos modificados o nuevo a el stagging area, todo archivo en stagging area formará parte del siguiente commit

Agregar un fichero indivdual al stagging area.

git add nombre_de_fichero.txt

Agregar ficheros modificados y hacer Commit con comentario

git commit -a -m "nuestro comentario sobre el commit" El parametro -a indica que se ejecute el comando git add previamente, así no hay que hacerlo en 2 pasos, pero no agregará ficheros untracked, es decir agregará y hará commit de los ficheros modificados que existían previamente en el anterior commit. Si tenemos ficheros nuevos tendremos que hacer 2 pasos, git add y git commit

Git status , conocer el estado de nuestro repositorio.

git status Nos indica cuantos commits por delante o por detrás vamos de la rama principal de desarrollo, además indica que ficheros se han modificado y cuales estan untracked ( ficheros nuevos ), Muy útil para ver lo modificado y decidir que agregaremos en el siguiente commit y que cambios echaremos para atrás

Git log , mostrar el log local de nuestro repositorio.

git log Si el log es largo se mostrará através del comando more, para movernos por el log, usamos f (forward) para ir hacia adelante b (backwards) para ir hacia atrás y q (quit) para salir del log Fuentes: Git ready - Git ref
